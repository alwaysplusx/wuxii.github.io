Cordova Plugin开发(Android)

发表于 2017-12-25 |

开发环境准备

  1. Android Studio
  2. Nodejs + Cordova

下载并安装安卓开发环境, 完成Cordova的基础环境配置
https://developer.android.com/studio/index.html
https://nodejs.org/en/
http://cordova.apache.org/

Cordova环境安装

  1. 安装Nodejs
  2. 全局安装cordova npm install -g cordova
  3. 尝试创建第一个Cordova应用 cordova create MyFirstApp

http://cordova.apache.org/docs/en/latest/guide/cli/index.html

注意: 如果使用cnpm安装可能导致cordova虽安装成功但不可用.
建议将npm的registry修改为淘宝镜像而不是使用cnpm来安装
npm config set registry https://registry.npm.taobao.org

插件环境

安装plugman

plugman是cordova提供的插件管理命令行工具, 不仅能用来创建插件的目录与文件还能提供丰富的插件管理功能.
http://cordova.apache.org/docs/en/latest/plugin_ref/plugman.html

创建插件工程

cordova plugin.xml

plugin.xml可以配置一下信息(包括不限于):

  • 开发者信息
  • 插件所支持的平台
  • 平台的源文件映射
  • 平台的依赖框架/lib映射
  • users-permission

创建成功后注意修改对应的映射关系, 将java的包名修改为可用的包名

http://cordova.apache.org/docs/en/latest/plugin_ref/spec.html

基础的插件框架结构完成后就可以生成对应的插件package.json了.

一个可被引入项目的插件壳就完成了.

插件功能开发

将做好的插件壳通过插件的方式引入到具体的项目中进行开发是一种较为可行的方法.

插件引入后即可进入具体的插件功能开发, 由于各个平台的插件与平台库有较强的依赖性, 所以需要生成对应的项目才能对插件功能进行开发. 将以Android为例, 进行插件功能的开发.

对应的插件代码也将一同导入到项目中, 可以在项目中对插件代码自由编辑, 这样就能开发插件功能了.
