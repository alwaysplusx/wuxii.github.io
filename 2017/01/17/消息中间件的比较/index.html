发表于 2017-01-17 |

Java MessageDriven中间件的比较

配置对比

Weblogic

Weblogic中的JMS配置由几部分构成

  1. JMS服务器
    1.1. 包含持久性存储(文件/JDBC)
    1.2. 目标(指向应用服务的Server/AdminServer)
  2. JMS模块(包含连接工厂/队列/主题等等)

不支持动态创建JMS资源
JMS服务器中建立JDBC持久性存储时候选择的数据源必须是非XA数据源
JMS模块中创建队列/主题时通过选择子部署来决定连接到具体的JMS服务器
注意:
weblogic下设置jms属性需要符合java命名规范Character#isJavaIdentifierPart

Glassfish

  1. 物理存储
    1.1. 只提供了文件类型的物理存储
    1.2. 配置方式(web-console/server(管理服务器)/JMS物理目标)
  2. JMS资源(连接工厂/队列/主题)
    2.1. 通过JMS物理目标的名称配置指定JMS物理目标

可以通过连接工程创建JMS资源(队列/主题), 名称不允许/分割

ActiveMQ

  1. brokerService
  2. persistenceAdapter
  3. connectors
  4. jmx/jolokia REST Api
  5. transaction
