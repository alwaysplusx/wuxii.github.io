Maven常用命令与插件

发表于 2016-12-30 |

Apache Maven是当今主流的Java自动化构建工具, 在Java开发人员中属于必备技能之一. maven的插件架构使得所有的功能都通过插件来完成, 了解基础命令以及常用插件成了maven学习的重中之重.

常用命令

Maven常用命令一览:

  • mvn -D
    别名: mvn –define
    说明: 定义系统参数
    示例:

  • mvn -f
    别名: mvn –file
    说明: 指定pom.xml文件(默认为当前目录中的pom.xml)
    示例:

  • mvn -v
    别名: mvn –version
    说明: 显示maven版本

Maven常用插件

super pom中已经存在的插件

  • maven-antrun-plugin
  • maven-assembly-plugin
  • maven-dependency-plugin
  • maven-release-plugin
  • maven-source-plugin
  • maven-javadoc-plugin
  • maven-deploy-plugin

maven-archetype-plugin(构建)

常用命令:

  • mvn archetype:generate
    说明: 按模版创建maven工程
  • mvn archetype:create
    说明: 直接创建maven工程

maven-compiler-plugin(编译)

常用命令:

  • mvn compile
    说明: 编译源文件(默认编译src/main/java下源文件到target/classes, 并拷贝src/main/resources下文件到target/classes下)
  • mvn compile:testCompile
    说明: 编译测试用例

maven-clean-plugin(清理)

常用命令:

  • mvn clean
    说明: 清除编译后文件(删除${base.dir}/target下的所有文件)

maven-dependency-plugin(依赖分析)

说明: maven依赖分析插件
常用命令:

  • mvn dependency:analyze
    说明: 识别哪些依赖有使用哪些无需使用
  • mvn dependency:tree
    说明: 树形展示项目依赖的jar包
  • mvn dependency:list
    说明: 列出项目的所有依赖

maven-resources-plugin(资源拷贝)

常用命令:

  • mvn resources:resources
    说明: 把main下的文件拷贝到main输出目录中
  • mvn resources:testResources
    说明: 把test下的文件拷贝到test输出目录中

maven-surefire-plugin(测试)

常用命令:

  • mvn test / mvn surefire:test
    说明: 运行测试用例
  • mvn surefire-report:report
    说明: 生成测试报告

maven-jar-plugin(打包)

在maven的package生命周期中会自动使用已经定义的打包方式, packaging为jar的或未填写的默认会打成jar包

常用命令:

  • mvn jar
    说明: 打jar包
  • mvn package
    说明: 打包文件(打包前会自动运行测试用例, 可使用-Dmaven.test.skip=true跳过).

maven-install-plugin(安装到repository)

常用命令:

  • mvn install
    说明: 安装到本地repository. (安装时会触发package, 而后将打包好的包拷贝到本地repository中, 存放的目录结构为${repository}/${groupId}/artifactId/version/artifactId-version.packaging)
  • mvn install:install-file
    说明: 安装本地jar文件到repository

maven-javadoc-plugin(项目文档api)

常用命令:

  • mvn javadoc:javadoc
    说明: 生成项目文档api
  • mvn javadoc:test-javadoc
    说明: 生成测试用例文档api
  • mvn javadoc:jar
    说明: 将api打包

maven-site-plugin(项目站点)

常用命令:

  • mvn site:site
    说明: 生成项目站点

maven-deploy-plugin(软件包发布)

将本地发布包发布到repository服务器上, 此时需要注意如果未在pom.xml文件中配置distributionManagement则效果同mvn install. 当需要验证权限时在settings.xml中配置deploy repository的安全策略(一般为用户名密码). settings.xml中配置的server id需要与pom.xml中的distributionManagement中的id相对应.

常用命令:

  • mvn deploy
    说明: 将打好的包发布到远程repository中去.
  • mvn deploy:deploy-file
    说明: 指定文件部署到远程库中

maven-eclipse-plugin(eclipse辅助插件)

常用命令:

  • mvn eclipse:eclipse
    说明: 生成eclipse工程文件
  • mvn eclipse:clean
    说明: 清除eclipse配置文件
