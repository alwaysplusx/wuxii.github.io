Maven基础介绍

发表于 2016-12-29 | 分类于 maven |

Apache Maven?

Apache Maven是用于Java项目自动化构建的工具, 其主要有两个目的, 其一描述项目是如何构建的; 其二描述项目的整体依赖. 工程主要通过XML文件描述其构建过程以及构建所需要的依赖.

Maven自动从Maven Central Repository下载Java libraries和相关插件包. Maven采用插件架构, 用户可以根据自己的需求独立开发相应的插件.

约定优于配置

Maven从许多项目实践中得出一套约定的配置, 在用户没有定制需求之前所有的基础配置都采用约定配置展开. 如在未做定制时项目的结构如下:

目录 说明
${basedir} 项目根目录, 存放pom.xml和所有子文件与目录
${basedir}/src/main/java Java源文件
${basedir}/src/main/resources 资源文件
${basedir}/src/test/java 测试类代码
${basedir}/src/test/resources 测试使用的资源
${basedir}/targets 编译后文件存放的根目录

生成的项目结构

pom.xml

pom.xml文件用于描述工程的构建过程以及工程的以来关系. 其位于项目的根目录下, 典型文件格式如下:

GAV

  • groupId: 组
  • artifactId: 项目名称
  • version: 项目版本

Maven采用GAV来确定一个项目(还存在细分的配置项如:classfilter, type), 在Maven中央库中对项目也是如此定位的.

版本号的一般规则为: X.Y.Z-[T]

  • X - 大版本号, 用于表示重大更新或对兼容有较大影响的更新
  • Y - FIX版本, 允许新特效添加但对兼容性影响不大
  • Z - BugFix版, 仅对bug进行必要修复
  • T - 版本名称, 可选.

依赖管理

在pom.xml dependencies节点下添加依赖, 如果需要排除直接依赖的子依赖查看如下说明:

依赖Scope

  • compile(default) - 编译范围
    此依赖在所有的classpath 中可用，同时它们也会被打包

  • provided - 已提供
    依赖已被提供在运行的容器中或jdk提供.如项目编译时classpath需要servlet-api, 但是不需要打包在发布包中,因为此servlet-api本因由应用服务器提供.此时即可采用provided

  • runtime - 运行时
    依赖在运行和测试系统的时候需要, 但在编译的时候不需要. 比如在编译的时候只需要JDBC API, 而在运行时才需要JDBC
    驱动实现。

  • test - 测试范围
    仅在测试编译和运行时需要

  • system - 系统
    与provided类似, 但必须提供一个本地系统的jar文件路径(maven不会去中央库中寻找).

Maven 配置(settings.xml)

settings.xml为Maven的全局配置文件, 其默认位于${MAVEN_HOME}/confg/settings.xml, 建议将文件拷贝至用户目录${user_home}/.m2下作为当前用户的配置.

Lifecycle Description
validate 检验项目是否正确配置以及项目的资源处于可用状态
initialize 初始化构建, 如.设置构建属性以及目录创建
generate-sources 生成编译所需要的代码
generate-resources 生成/拷贝资源文件
compile 编译源码
test-compile 编译测试用例源码
test 运行测试用例
package 将编译后的文件打包成发布文件格式(such as a JAR)
install 安装发布包到本地的repository
deploy 将发布包发布到私服供其他项目使用
