JavaEE版本历史

发表于 2016-12-28 | 分类于 javaee |

JavaEE在1.4版本前称为J2EE, 1.5的发布(也就是JavaEE 5)正式更名为JavaEE.

JavaEE 5

J2EE的糟糕开发体验使得Spring/Struts/Hibernate得以大行其道.为了解决笨重的J2EE问题,在2006年推出了JavaEE 5, 其中EJB 3.0有了长足的进步,JPA的出现也使得实体存储的标准得到统一.

SOAP with Attachments API for Java (SAAJ) 1.3
Streaming API for XML (StAX) 1.0

JavaEE 6

由JavaEE 6开始JavaEE规范开始整和来自Spring的概念与特性. JavaEE 6的轻量级优化与删减了大部分未使用的API, 使得JavaEE规范得到了较大范围的流行.

Spring IoC -> Dependency Injection for Java
Spring AOP -> Interceptors
Spring Beans -> Session Bean

Web Application Technologies:

名称 版本
* Java Servlet 3.0
JavaServer Faces (JSF) 2.0

名称 版本

* Java Persistence API (JPA) 2.0

* Dependency Injection for Java 1.0
* Bean Validation 1.0
Managed Beans 1.0
* Interceptors 1.1

JavaEE 7

跟随时代的变化JavaEE 7开始拥抱HTML5/WebSocket/RESTful. 为大数据时代准备了平台级的并发与批处理规范.
